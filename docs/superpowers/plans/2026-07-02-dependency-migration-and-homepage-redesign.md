# Dependency Migration and Homepage Redesign Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade every dependency in this Next.js/T3 app to its latest version, then replace the site's generic gradient-and-glow visual language with a sage-mint brutalist look across the shared header and every signed-out entry screen.

**Architecture:** Two sequential parts. Part A is a dependency bump (`pnpm up --latest`) followed by an iterative fix-the-breakage loop verified by `pnpm typecheck` / `pnpm lint` / `pnpm build`. Part B is a CSS token swap in `globals.css` plus a rewrite of four small screen files to the approved brutalist/sage-mint direction, using shadcn's existing CSS-variable system so `Button` and other primitives pick up the new palette automatically.

**Tech Stack:** Next.js (App Router), TypeScript, Tailwind CSS v4, ShadCN/UI (CVA + Radix Slot), Clerk, Drizzle ORM, pnpm.

## Global Constraints

- No em dashes or hyphens used as punctuation anywhere written: commit message, UI copy, code comments, this plan's own prose already follows this. Regular kebab-case identifiers, CSS properties, and `--flag` syntax are fine, those aren't prose.
- This plan executes in an isolated worktree/branch. Each task commits normally there (needed for per-task review). When the branch merges to `main`, it gets squashed into exactly **one** commit. That squash commit message must be 8 to 11 words, terse and slogan-like, no "Co-Authored-By" trailer. Task 9's "single commit" step below refers to this final squash, not a same-worktree constraint on Tasks 1 to 8.
- Production has real user data. Nothing in this plan touches `src/server/db/schema.ts` or runs `pnpm db:push`. If a dependency bump seems to demand a schema or data change, stop and flag it instead of proceeding.
- Full spec: `docs/superpowers/specs/2026-07-02-dependency-migration-and-homepage-redesign-design.md`
- Addendum (added after Tasks 1 to 9 were done and reviewed): the user asked to also restyle the authenticated file browser (`src/components/drive-content.tsx`, `src/components/file-row.tsx`) to match a reference mockup, reversing the earlier out-of-scope call. Task 10 below covers this. The reference is a small stacked, rotated, monospace file-list card (folder rows in amber, a file row with its type icon) on a near-black card, the same "stacked photo" motif as the mockups from brainstorming, now applied to the real, functional file list instead of just a static preview.

- Modify: any other file `pnpm typecheck` or `pnpm lint` flags

**Interfaces:** none, this task only fixes existing call sites to match new library APIs, it doesn't introduce new exports other code depends on.

This task is inherently exploratory since the exact breaking changes ship with each release. Work in this loop:

- [ ] **Step 1: Run typecheck and capture the first batch of errors**

Run: `pnpm typecheck`

- [ ] **Step 2: For each distinct error, identify the offending package and look up its migration notes**

If an error traces to `zod` (likely in `src/env.js`, which uses `z.string().url()` and `z.enum(...)`), use WebSearch for "Zod 4 migration guide breaking changes" and fix the call sites. A common Zod 4 change: string format validators like `.url()` move to top-level functions (`z.url()`); confirm the current API before editing rather than guessing.

If an error traces to `@clerk/nextjs`, use WebSearch for "Clerk Next.js v7 migration guide" and check whether `clerkMiddleware()` in `src/middleware.ts:1-3` or the `auth()` calls in `src/app/(home)/page.tsx`, `src/app/(home)/sign-in/page.tsx`, `src/app/(home)/drive/page.tsx` need signature changes.

If an error traces to `lucide-react`, use WebSearch for "lucide-react v1 migration breaking changes" and fix the imports in `src/components/file-row.tsx:1-16` and any other file importing icons (`src/app/(home)/page.tsx`, `src/app/layout.tsx` if it imports icons after Part B rewrites it, `src/components/drive-content.tsx`).

If an error traces to `typescript-eslint` / `eslint` / `eslint-config-next`, use WebSearch for "ESLint 10 flat config migration" and "eslint-config-next 16 breaking changes", then adjust `eslint.config.js`.

If an error traces to `drizzle-orm` or `drizzle-kit`, use WebSearch for "drizzle-orm singlestore breaking changes" for the specific version jump recorded in Task 1 Step 4, since `src/server/db/schema.ts` uses `singlestoreTableCreator`, `bigint`, `index`.

- [ ] **Step 3: Apply the fix, then re-run typecheck**

Run: `pnpm typecheck`
Repeat Steps 2 to 3 until it exits clean.

- [ ] **Step 4: Run lint and fix what it flags**

Run: `pnpm lint`

Most flat-config or rule-name issues surface here rather than in typecheck. Fix and re-run until clean. If `eslint-config-next` 16 renamed or removed a rule referenced in `eslint.config.js:14` (`next/core-web-vitals`), follow its migration notes rather than deleting the rule silently.

- [ ] **Step 5: Confirm no fix required a database or schema change**

Read through the diff so far: `git diff -- src/server/db/`
Expected: empty, or only whitespace/formatting from an ORM version bump touching generated types, never a change to table/column definitions. If it's not empty in a meaningful way, stop and flag it per the Global Constraints rather than proceeding.

- [ ] **Step 6: Commit Task 1 and Task 2 together**

Task 1's dependency bump and Task 2's fixes are one unit, an upgrade alone is not a working state, so they land in a single commit:


```tsx
export function FolderRow(props: {
  folder: typeof folders_table.$inferSelect;
  last?: boolean;
  isEditing?: boolean;
  onEditComplete?: () => void;
}) {
  const { folder, last = false, isEditing = false, onEditComplete } = props;
  const [isEditMode, setIsEditMode] = useState(isEditing);
  const [folderName, setFolderName] = useState(folder.name);
  const inputRef = useRef<HTMLInputElement>(null);

  const [prevIsEditing, setPrevIsEditing] = useState(isEditing);
  const [prevFolderNameProp, setPrevFolderNameProp] = useState(folder.name);
  if (isEditing !== prevIsEditing || folder.name !== prevFolderNameProp) {
    setPrevIsEditing(isEditing);
    setPrevFolderNameProp(folder.name);
    if (isEditing) {
      setIsEditMode(true);
      setFolderName(folder.name);
    }
  }

  useEffect(() => {
    if (isEditMode && inputRef.current) {
      inputRef.current.focus();
      inputRef.current.select();
    }
  }, [isEditMode]);

  const handleSave = async () => {
    if (folderName.trim() === "") {
      setFolderName(folder.name);
      return;
    }

    try {
      const result = await RenameFolder(folder.id, folderName.trim());
      if (result.error) {
        console.error("Failed to rename folder:", result.error);
        setFolderName(folder.name); // Revert to original name
      }
    } catch (error) {
      console.error("Error renaming folder:", error);
      setFolderName(folder.name); // Revert to original name
    }

    setIsEditMode(false);
    onEditComplete?.();
  };

  const handleCancel = () => {
    setFolderName(folder.name);
    setIsEditMode(false);
    onEditComplete?.();
  };
  const handleKeyDown = (e: React.KeyboardEvent) => {
    if (e.key === "Enter") {
      void handleSave();
    } else if (e.key === "Escape") {
      handleCancel();
    }
  };

  return (
    <li
      key={folder.id}
      className={`px-6 py-4 font-mono text-sm hover:bg-accent/50 ${last ? "" : "border-b border-border"}`}
    >
      <div className="grid grid-cols-12 items-center gap-4">
        <div className="col-span-6 flex items-center">
          {isEditMode ? (
            <div className="flex w-full items-center">
              <FolderIcon className="mr-3 text-amber-400" size={20} />
              <input
                ref={inputRef}
                type="text"
                value={folderName}
                onChange={(e) => setFolderName(e.target.value)}
                onKeyDown={handleKeyDown}
                className="w-2/3 border border-border bg-input px-2 py-1 font-mono text-foreground focus:border-primary focus:outline-none"
              />
              <div className="ml-2 flex gap-1">
                <Button
                  variant="ghost"
                  size="sm"
                  onClick={() => void handleSave()}
                  className="h-6 w-6 cursor-pointer p-0 hover:bg-primary/20 hover:text-primary"
                >
                  <CheckIcon size={14} />
                </Button>
```
